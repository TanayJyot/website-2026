# abstract data type
## [[10.4, 10.5]]
- The idea is that there are some common functions of some common data types in every language like how we call any kind of a table based data storage an [[array]]. 
- We only care about the basic [[Operations]] all of their versions perform and what they are. We do not care about how they work. 
- Here it is the idea that matters not an actual concrete type. Unlike [[concrete data type]]
```python
# A Mapping using a Python dict
{0: 'hello', 1: 42, 2: 'goodbye'}

# A Mapping using a Python list
[(0, 'hello'), (1, 42), (2, 'goodbye')]
```

- The reason one [[concrete data type]] would be better than another at implementing an [[abstract data type]] comes down to [[Efficiency]]. 
- contains data stored and the operations that can be performed
